4. How the Extension Works

The Extension operates entirely within your browser:

  1. Uses Your Existing Session: The Extension uses your existing PPAS login session (browser cookies) to authenticate API requests. It does NOT access, store, or transmit your credentials.
  2. Sends Approval Requests: When you click "Start Approval," the Extension sends POST requests to ppas.pdsodisha.gov.in on your behalf, just like manual approval.
  3. Processes Responses Locally: Server responses (HTML data) are parsed locally in your browser using JavaScript's DOMParser. No data is sent to external servers.
  4. Displays Results: The Extension shows progress and results directly in the popup interface. All processing happens client-side.

6. Permissions Explained

The Extension requests the following Chrome permissions:

6.1. "scripting" Permission

Why needed: To inject automation code into PPAS web pages for interacting with token approval forms and parsing HTML responses.

Privacy impact: This permission allows the Extension to read and modify content on ppas.pdsodisha.gov.in pages only. It does NOT allow access to other websites.

6.2. "activeTab" Permission

Why needed: To access the currently active PPAS tab when you click "Fetch Tokens" or "Start Approval" buttons.

Privacy impact: This permission only grants temporary access to the active tab and only when you explicitly trigger an action. The Extension cannot monitor your browsing in the background.

6.3. "storage" Permission

Why needed: To save your preferences (PACS, PPC, date, token limit) locally using chrome.storage.local API.

Privacy impact: Data is stored only on your device and never synchronized or transmitted. You can clear this data anytime by clearing browser data.

6.4. Host Permission: "https://ppas.pdsodisha.gov.in/*"

Why needed: To send API requests to PPAS portal endpoints for fetching and approving tokens.

Privacy impact: This restricts the Extension to ONLY work on the PPAS portal. It cannot access any other websites or domains.
